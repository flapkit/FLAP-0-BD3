# Contributing to FLAP-0 BD3

Thank you for your interest in contributing to FLAP-0 BD3! This document provides guidelines and instructions for contributing to the project.

## Code of Conduct

This project adheres to a Code of Conduct that all contributors are expected to follow. Please read [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) before contributing.

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report bugs, suggest features, or ask questions
- Check existing issues to avoid duplicates
- Provide clear descriptions and steps to reproduce (for bugs)
- Include relevant hardware/software versions and environment details

### Contributing Hardware Design Changes

1. **Fork the repository** and create a feature branch
2. **Make your changes** to the hardware design files
3. **Update documentation** as needed (schematics, BOM, assembly instructions)
4. **Test your changes** if possible
5. **Submit a pull request** with a clear description of changes

### Contributing Documentation

- Fix typos, improve clarity, or add missing information
- Follow the existing documentation style
- Update related documentation when making changes
- Include diagrams or images where helpful

### Design File Guidelines

- Maintain editable source formats (KiCad, Altium, etc.)
- Export PDF versions for review
- Update BOM when component changes are made
- Document any design decisions or trade-offs

### Pull Request Process

1. Ensure your changes follow the project structure
2. Update CHANGELOG.md with your changes
3. Ensure all files are properly licensed
4. Request review from maintainers
5. Address any feedback before merging

## Development Setup

1. Clone the repository
2. Review the hardware design files in `hardware/bd3_rev01/`
3. Check manufacturing outputs in `manufacturing/bd3_rev01_release/`
4. Refer to documentation in `docs/`

## Questions?

If you have questions about contributing, please:
- Open an issue with the "question" label
- Contact the maintainers at info@flapkit.org
